# Conflict Rules

Conflict handling keeps the wiki layer trustworthy without hiding uncertainty.

## Duplicate Pages

Before creating a new wiki page:

1. Search existing `wiki/` pages by title, slug, and key terms.
2. Update an existing page when the topic is the same.
3. Create a new page only when the new topic has a distinct purpose or audience.

## Stale Pages

A page is stale when:

- Its sources have been superseded by newer raw sources.
- Its conclusions conflict with newer cited material.
- Its `updated_at` is old and the topic changes frequently.

When a page is stale, update it in place unless the old state is historically important. If history matters, create or update a timeline page.

## Conflicting Claims

When sources disagree:

1. Keep both claims visible.
2. Cite the raw source behind each claim.
3. Explain the difference in scope, date, version, author, or assumptions when possible.
4. Do not collapse disagreement into a false consensus.

## Uncertain Information

Use explicit uncertainty instead of confident language when raw support is weak:

- `Unclear:` for missing evidence.
- `Tentative:` for plausible but not fully supported synthesis.
- `Conflict:` for directly contradictory sources.

## Deletion

Do not delete sourced wiki content just because it is old. Prefer marking it stale, superseded, or historically scoped.
